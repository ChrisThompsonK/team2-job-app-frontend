#### Testing Your Changes

The project includes comprehensive unit and E2E testing with automatic report generation.

##### Running Tests

```bash
# Unit tests (watch mode)
npm run test

# Unit tests (one-time run with report)
npm run test:run

# Unit tests with coverage metrics
npm run test:coverage

# E2E tests
npm run test:e2e

# View latest test report
npm run report:view
```

##### Test Reports

After running tests with `npm run test:run`, `npm run test:coverage`, or `npm run test:e2e`, a detailed test report is automatically generated in the `reports/` directory.

Reports include:
- Overall test status (pass/fail)
- Test count and pass rate
- Code coverage metrics (lines, branches, functions, statements)
- Failed test details with error messages
- Test execution time

View the latest report:
```bash
npm run report:view
```

**Note**: Reports are saved in `reports/test-report-latest.txt` and `reports/test-report-latest.json`

##### Coverage Requirements

- **New Code**: Minimum 80% coverage
- **Controllers**: 100% branch coverage
- **Services**: 100% branch coverage for critical paths
- **Overall**: Maintain or improve existing coverage

For full testing documentation, see [Testing Quick Start Guide](./TESTING-QUICK-START.md)
```
